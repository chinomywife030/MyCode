# 🧪 UI 優化驗收測試指南

## 📋 測試目標

驗證身份模式切換的 UI/UX 優化是否符合需求：
1. 使用者一眼知道目前身份模式
2. 切換按鈕足夠大，適合手機操作
3. 色彩切換流暢，不混色

---

## 🔍 測試步驟

### A. 模式切換按鈕測試

#### 1. 視覺檢查
- [ ] **桌面版**：Header 上的切換按鈕清晰可見，高度 ≥ 44px
- [ ] **手機版**：切換按鈕高度 ≥ 44px，寬度 ≥ 90px
- [ ] **Active 狀態**：
  - 需求者：藍色實心背景 + 白字 + 陰影 + 輕微放大
  - 代購者：橘色實心背景 + 白字 + 陰影 + 輕微放大
- [ ] **Inactive 狀態**：灰色文字 + 淡背景
- [ ] **Hover 狀態**：有視覺反饋（背景變白/灰）

#### 2. 功能測試
```
1. 打開首頁
2. 點擊「需求者」按鈕
   ✅ 按鈕變藍色實心
   ✅ Logo 變藍色
   ✅ Hero 背景變藍色漸層
   ✅ 主按鈕文字變「發佈需求」
3. 點擊「代購者」按鈕
   ✅ 按鈕變橘色實心

   ✅ Hero 背景變橘色漸層

4. 重複切換 3 次
   ✅ 每次切換都有 0.2s 流暢動畫
   ✅ 無閃爍或卡頓
```

#### 3. 手機單手操作測試
```
📱 使用手機（或 Chrome DevTools 手機模擬）
1. 單手持手機
```

## 🐛 常見問題排查

### 問題 1：切換後顏色沒變
**檢查**：
- `UserModeProvider` 是否正確包裹 App
- `localStorage` 是否正常讀寫
- 瀏覽器是否阻擋 localStorage

### 問題 2：切換動畫卡頓
**檢查**：
- 是否有 `transition-all duration-200`
- 是否有過多的 re-render
- 瀏覽器性能是否正常

### 問題 3：手機上按鈕太小
**檢查**：
- `minHeight` 是否 ≥ 44px
- `padding` 是否足夠
- 是否有 `cursor: pointer`

### 問題 4：刷新後模式重置
**檢查**：
- `localStorage.getItem('bangbuy_mode')` 是否有值
- `UserModeProvider` 初始化邏輯是否正確

- [x] 跨頁面/刷新後模式保持
- [x] 所有按鈕有 `cursor: pointer` 和 `aria-*` 屬性
